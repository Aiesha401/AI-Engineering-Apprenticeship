# Day 9

## What did I build?

I extended my chatbot to support tool arguments using the official Function Calling API. Instead of calling a function with no parameters, my chatbot now allows the LLM to generate JSON arguments, which Python parses and passes to the appropriate function.

## What did I learn?

Today I learned how function parameters are described using a JSON schema inside the `tools` definition. I also learned that the LLM not only decides which function to call but also extracts the required arguments from natural language and returns them as a JSON string. I used `json.loads()` to convert the JSON string into a Python dictionary before passing the arguments to my function.

## Biggest challenge today

Understanding why my function raised a `TypeError` for a missing parameter and figuring out how to pass the arguments generated by the LLM into the Python function.

## How did I solve it?

I inspected the `tool_calls` response and discovered that the arguments were returned as a JSON string. After converting them into a Python dictionary using `json.loads()`, I extracted the `product` value and passed it into the `get_inventory()` function.

## What surprised me?

I was surprised that the LLM could automatically extract structured information like `"product": "iPhone 16"` from a natural language question without me writing any parsing logic or regular expressions.

## One thing I still don't understand

I'm curious to learn how the LLM decides whether to generate a normal text response or request one or more tool calls, and how applications handle both situations cleanly.
