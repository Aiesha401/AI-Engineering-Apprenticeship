# Day 13

## What did I build?

Today I practiced writing SQL queries against the inventory table I created yesterday. I retrieved specific columns, filtered data with WHERE, sorted results using ORDER BY, limited the number of returned rows with LIMIT, and combined these clauses into more advanced queries.

## What did I learn?

I learned how to ask meaningful questions of a database instead of simply retrieving all data. I practiced selecting only the information I needed, filtering records based on conditions, sorting results in ascending and descending order, and limiting the number of returned rows. I also learned that SQLite is more permissive with string quoting than many other databases, but using single quotes for string literals is the standard SQL convention.

## Biggest challenge today

The biggest challenge was learning to think in SQL rather than memorizing syntax. Instead of focusing on keywords, I had to translate plain English questions into SQL statements.

## How did I solve it?

I solved this by completing multiple exercises that gradually combined different SQL clauses. Repeating the same Python structure while changing only the SQL query helped me focus on understanding what each clause was doing.

## What surprised me?

I was surprised by how readable SQL is. Complex queries can almost be read like English, which made it easier to understand and remember the syntax than I initially expected.

## One thing I still don't understand

I'm curious about how developers inspect and explore large databases in practice without remembering every table and column name. I'd like to learn the tools and SQL commands used to view database schemas efficiently.
